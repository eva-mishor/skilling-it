# End-of-Day Summary Generator

This command creates a comprehensive daily work summary combining automated metrics with personal reflection.

## Phase 1: Automated Metrics Collection

### 1.1 Git Activity Analysis

Analyze today's git activity:

```bash
# Get today's commits with author filter
git log --since="midnight" --author="$(git config user.name)" --pretty=format:"%h - %s" --no-merges

# Get detailed statistics
git log --since="midnight" --author="$(git config user.name)" --stat --pretty=format:"" --no-merges | tail -n 1

# Check for uncommitted changes
git status --short
```

**Extract:**
- Commit count and messages
- Files changed (grouped by component: algorithms/, cli/, compressor/, tests/, scripts/, docs/)
- Lines added/removed per component
- Uncommitted work status

### 1.2 Project-Specific Metrics

**Database Metrics:**
```bash
# Find database files
find . -name "*.db" -not -path "*/venv/*" -not -path "*/.pytest_cache/*" 2>/dev/null

# For each database, get size and basic stats
ls -lh [database-file]
```

**Dataset Coverage:**
```bash
# Count files in data directories
find data/ -type f 2>/dev/null | wc -l

# Get folder structure
find data/ -type d -maxdepth 2 2>/dev/null
```

**Test Coverage:**
```bash
# Run quick coverage check (if possible without breaking workflow)
pytest --cov=compressor --cov-report=term-missing --quiet 2>/dev/null | grep "TOTAL" || echo "Coverage check skipped"
```

### 1.3 Todo.md Analysis

**Scan for completed tasks:**
```bash
# Read todo.md and identify completed tasks
grep -A 1 "status.*completed" todo.md || echo "No completed tasks found"
```

**Identify:**
- Tasks marked as completed today
- Tasks still in-progress
- New tasks added

## Phase 2: Interactive Reflection

Present the automated metrics summary, then prompt for reflection:

### Reflection Questions:

**Ask the user:**

1. **🎯 What were your 3 main accomplishments today?**
   - (Focus on concrete deliverables, features, fixes, or insights)

2. **💡 What technical insight or learning stood out today?**
   - (New technique, library, pattern, or "aha!" moment)

3. **🚧 What challenges or blockers did you face?**
   - (Problems encountered, external dependencies, unclear requirements)

4. **📅 What's your top priority for tomorrow?**
   - (Next task to tackle, what needs focus)

5. **🔄 What would you do differently or improve?**
   - (Process, approach, time management, communication)

**Note:** If argument `--metrics-only` or `--quick` is provided, skip reflection questions.

## Phase 3: Generate Daily Log

### 3.1 Create Log File

Create file: `.claude/logs/daily/YYYY-MM-DD.md`

**Template:**

```markdown
# Daily Summary - [Full Date]

Generated at: [HH:MM]

---

## 📊 Git Activity

### Commits Today: [N]
[List each commit with hash and message]

### Files Changed: [N]
**By Component:**
- algorithms/: [N files] (+X/-Y lines)
- cli/: [N files] (+X/-Y lines)
- compressor/: [N files] (+X/-Y lines)
- tests/: [N files] (+X/-Y lines)
- scripts/: [N files] (+X/-Y lines)
- docs/: [N files] (+X/-Y lines)

**Total:** +[XXX]/-[YYY] lines

### Uncommitted Changes
[List if any, or "None"]

---

## 🗄️ Project Metrics

### Database
- **Primary DB:** [filename] ([size])
- **Total Rows:** [count across all tables]
- **Recent Operations:** [if available]

### Datasets
- **Total Files:** [N] files
- **Categories:** [list folders in data/]

### Test Coverage
- **Current:** [X]%
- **Change:** [+/-Y%] from last measurement

---

## ✅ Tasks Completed Today

[Auto-extracted from todo.md completed tasks]

---

## 🎯 Accomplishments

[User's response to question 1]

---

## 💡 Learning & Insights

[User's response to question 2]

---

## 🚧 Challenges & Blockers

[User's response to question 3]

---

## 📅 Tomorrow's Focus

[User's response to question 4]

---

## 🔄 Process Reflection

[User's response to question 5]

---

## 📝 Additional Notes

[Optional: any additional notes provided as command argument]

---

*Generated by /end-of-day command*
```

### 3.2 Update todo_completed.md

**If completed tasks found:**

1. Read current `todo_completed.md`
2. Append new section:
```markdown
## Completed - [YYYY-MM-DD]

[List of tasks marked completed today from todo.md]
```

3. Update `todo.md` to remove completed tasks from active list

### 3.3 Display Summary

Show user:
- Location of saved log file
- Count of completed tasks moved
- Summary stats (commits, files, coverage)
- Prompt to review the generated log

## Usage Examples

```bash
# Full interactive mode with all prompts
/end-of-day

# Quick metrics-only mode (skip reflection)
/end-of-day --metrics-only

# With additional notes
/end-of-day "Focused on Phase 1A database optimization"

# Quick abbreviated mode
/end-of-day --quick
```

## Important Notes

- **Local Only**: All logs are stored in `.claude/logs/` which is gitignored
- **Privacy**: Reflection answers stay local, never committed to repository
- **Automation**: Completed tasks automatically archived to `todo_completed.md`
- **Flexibility**: Skip reflection questions if in a hurry with `--quick` flag

## Output Location

All daily logs saved to: `.claude/logs/daily/YYYY-MM-DD.md`

---

## Execution Flow

1. ✅ Collect automated git metrics
2. ✅ Gather project-specific metrics (DB, datasets, coverage)
3. ✅ Scan todo.md for completed tasks
4. ✅ Present metrics summary to user
5. ❓ Prompt for reflection questions (unless --quick)
6. 📝 Generate formatted markdown log
7. 🗂️ Update todo_completed.md with archived tasks
8. 📋 Display summary and log location
